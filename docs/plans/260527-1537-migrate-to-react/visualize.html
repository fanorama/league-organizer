Migrasi ke React

Memindahkan rendering layer League Organizer dari vanilla JS + innerHTML ke React (Vite + TypeScript + Zustand + React Router), tanpa mengubah business logic, storage schema, atau design system.

  • 5 fase
  • Vite + React 18
  • TypeScript
  • Zustand
  • React Router v6
  • localStorage kompatibel

Data Flow — Zustand Bridge Pattern

flowchart TD
    A[React Component] -->|"selector: s => s.leagues"| B[Zustand Store]
    A -->|"action: createLeague()"| B
    B -->|"getAll(KEYS.leagues)"| C[localStorage]
    B -->|"save(KEYS.leagues, data)"| C
    C -->|"JSON.parse"| B
    B -->|"set({ leagues })"| A

    style A fill:#d9f2ed,stroke:#0f766e
    style B fill:#fff7ed,stroke:#f59e0b
    style C fill:#f3f4f6,stroke:#9ca3af

Zustand hanya sebagai view cache — localStorage tetap source of truth. Setiap mutasi: panggil storage.ts → lalu set() untuk trigger re-render.

Risiko Utama

RisikoMitigasi

CSS class names berbeda setelah konversi ke JSX Copy verbatim dari innerHTML lama ke className

Query string → path params Semua link internal diupdate sekaligus di Phase 5

Yang Tidak Berubah

localStorage schema identik — Data lama tetap kompatibel. Tidak ada migrasi data.

  • styles/main.css — tidak dimodifikasi sama sekali
  • Business logic di schedule.js dan standings.js — hanya tambah TypeScript types
  • Storage layer pattern — save(), getAll(), remove() tetap sama
  • Data model — League, Team, Season, Match schema identik
  • File JS lama (js/*.js) — tidak dihapus, tetap ada sebagai fallback
